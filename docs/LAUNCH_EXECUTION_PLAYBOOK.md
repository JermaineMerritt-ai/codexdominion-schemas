## 8. CONTINGENCY PLANS

### SCENARIO 1: Website Crash

**Trigger:** Server downtime > 5 minutes

**Action:**
1. Engineering Lead: Switch to backup server (Azure failover)
2. Marketing Lead: Pause all paid ads
3. PR Lead: Post update on social: "We're experiencing high traffic! Back shortly."
4. Founder: Evaluate if delay is needed (if downtime > 30 minutes)

**Prevention:**
- Load testing pre-launch (simulate 1,000 concurrent users)
- Auto-scaling enabled (Azure/AWS)
- CDN for static assets (Cloudflare)

---

### SCENARIO 2: Payment Gateway Failure

**Trigger:** Stripe API errors or failed transactions

**Action:**
1. Engineering Lead: Check Stripe dashboard for incidents
2. Disable checkout if issue persists > 15 minutes
3. Social/email: "We're fixing a checkout issue. Back shortly!"
4. Offer alternative: Manual invoicing via PayPal (for urgent buyers)

**Prevention:**
- Test transactions pre-launch (10+ test purchases)
- Stripe webhook monitoring (real-time alerts)
- Backup gateway (PayPal Business) ready to activate

---

### SCENARIO 3: Negative Press or Social Backlash

**Trigger:** Viral negative post, critical article, or controversy

**Action:**
1. PR Lead: Monitor sentiment (tools: Brandwatch, Hootsuite)
2. Founder: Draft response (acknowledge, clarify, commit to action)
3. Post official response within 4 hours
4. Legal review if needed (defamation, factual errors)

**Prevention:**
- Crisis communication plan pre-approved
- Founder trained on media response
- Community guidelines enforced (ban toxic users)

---

### SCENARIO 4: Low Sales (< 10 purchases in first 48 hours)

**Trigger:** Sales below target
