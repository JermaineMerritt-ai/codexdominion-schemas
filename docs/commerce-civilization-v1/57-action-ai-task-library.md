## 1. Purpose of the Task Library

The Task Library is a **catalog of the most common tasks** users can ask for, organized by type with clear examples.

**Why This Matters:**
- Helps new users understand what's possible
- Provides clear request examples they can model
- Reduces ambiguity in task requests
- Shows the breadth of system capabilities
- Makes onboarding faster and smoother

**Who Uses This Library:**
- New users learning what to request
- Experienced users exploring new task types
- Team leaders training others
- Documentation writers creating guides

**How to Use This Library:**
- Browse task categories to see what's possible
- Copy example requests and adapt to your needs
- Use consistent phrasing for better results
- Combine multiple task types for complex workflows

## 2. Drafting Tasks

**Purpose:** Create new content from scratch

**What This Includes:**
- Emails, letters, messages
- Product descriptions
- Announcements, updates
- Scripts, dialogues
- Social media posts
- Welcome messages
- Introductions
- Simple documents

### Common Requests

**Emails:**
- "Draft an email"
- "Write a professional email"
- "Create a thank you email"
- "Draft a meeting invitation"

**Product Descriptions:**
- "Draft a product description"
- "Write a short description for this product"
- "Create a product summary"

**Announcements:**
- "Draft a simple announcement"
- "Write a team update"
- "Create an announcement about [topic]"

**Scripts:**
- "Draft a short script"
- "Write a simple dialogue"
- "Create a conversation script"

**Social Posts:**
- "Draft a social post"
- "Write a Facebook post about [topic]"
- "Create a LinkedIn update"

**Welcome Messages:**
- "Draft a welcome message"
- "Write a greeting for new customers"
- "Create an onboarding message"

### Examples with Context

**Example 1: Business Email**
```
Request: "Draft a short email introducing our new service to existing customers."

Output:
Subject: Introducing [Service Name]

Hi [Customer Name],

We're excited to announce the launch of [Service Name], designed to help you [benefit].

This new service offers:
• [Key feature 1]
• [Key feature 2]
• [Key feature 3]

You can get started at [link]. If you have questions, just reply to this email.

Best regards,
[Your Name]
```

**Example 2: Product Description**
```
Request: "Write a product description for a digital planner."

Output:
This digital planner helps you organize your day, week, and month in one convenient place. Features include daily pages with time blocks, weekly overview spreads, monthly calendars, and goal-setting templates. Compatible with popular note-taking apps, it's perfect for busy professionals who want to stay organized without carrying physical planners.
```

**Example 3: Team Announcement**
```
Request: "Create a simple announcement for our team meeting next Friday."

Output:
Team Meeting — Friday, January 10 at 2:00 PM

Join us for our monthly team meeting to discuss:
• Q4 progress review
• New project kickoff
• Team updates and wins

Location: Conference Room B (or join via Zoom link)

See you there!
```

### When to Use Drafting Tasks

✓ You need new content created from scratch  
✓ You have a clear purpose but need help with wording  
✓ You want a starting point to revise  
✓ You need consistent formatting across similar content

## 3. Summarizing Tasks

**Purpose:** Condense information into shorter, clearer formats

**What This Includes:**
- Bullet-point summaries
- Key point extraction
- Length reduction
- Main idea identification
- Executive summaries

### Common Requests

**Bullet Summaries:**
- "Summarize this in bullet points"
- "Create a bullet-point summary"
- "Extract main points as bullets"

**Key Points:**
- "Extract the key points"
- "What are the main ideas?"
- "Pull out the key takeaways"

**Length Reduction:**
- "Reduce this to [number] sentences"
- "Shorten this to one paragraph"
- "Condense this to 100 words"

**Executive Summaries:**
- "Create an executive summary"
- "Summarize this for leadership"
- "Give me the highlights"

### Examples with Context

**Example 1: Five Bullet Points**
```
Request: "Summarize this report in five bullet points."

Input: [3-page market research report]

Output:
• Market size expected to grow 15% annually through 2028
• Primary competitors are focusing on mobile-first solutions
• Customer acquisition costs have decreased 20% year-over-year
• Survey shows 85% satisfaction rate among current users
• Recommended focus areas: product differentiation and customer retention
```

**Example 2: Key Ideas**
```
Request: "Extract the key ideas from this paragraph."

Input: [Long paragraph about project management methodology]

Output:
• Projects should be broken into small, manageable tasks
• Regular check-ins prevent drift and misalignment
• Clear ownership improves accountability
• Documentation ensures continuity
```

**Example 3: Length Reduction**
```
Request: "Shorten this email to three sentences."

Input: [Long, rambling email]

Output:
We're moving the project deadline to March 15 to accommodate the new requirements. Please update your timelines and let me know if you have concerns. I'll send a detailed update by Friday.
```

### When to Use Summarizing Tasks

✓ You have too much information and need the essentials  
✓ You need to share key points with stakeholders  
✓ You want to make long content scannable  
✓ You need to extract actionable insights

## 4. Organizing Tasks

**Purpose:** Structure or reorganize content for clarity

**What This Includes:**
- Adding headings and sections
- Creating bullet lists
- Building checklists
- Reorganizing messy notes
- Adding logical structure
- Improving flow

### Common Requests

**Headings and Sections:**
- "Add headings to this"
- "Organize this into sections"
- "Create clear sections with titles"

**Bullet Points:**
- "Turn this into bullet points"
- "Convert this to a list"
- "Make this a bulleted outline"

**Checklists:**
- "Turn this into a checklist"
- "Create a checklist from these tasks"
- "Make this a step-by-step checklist"

**Reorganization:**
- "Reorganize these notes"
- "Clean this up and make it easier to read"
- "Structure this more logically"

### Examples with Context

**Example 1: Add Structure**
```
Request: "Organize this into sections with headings."

Input: [Unstructured paragraph about project phases]

Output:
## Planning Phase
We'll define objectives, identify stakeholders, and create a timeline.

## Execution Phase
The team will build deliverables according to the approved plan.

## Review Phase
We'll test results, gather feedback, and make final adjustments.

## Delivery Phase
Final products will be delivered to stakeholders with documentation.
```

**Example 2: Create Checklist**
```
Request: "Turn this into a checklist."

Input: "Before launching, make sure you've tested the links, reviewed the copy, checked mobile formatting, and confirmed the email addresses."

Output:
Launch Checklist:
☐ Test all links
☐ Review copy for errors
☐ Check mobile formatting
☐ Confirm email addresses
```

**Example 3: Clean Up Notes**
```
Request: "Clean this up and make it easier to read."

Input: [Messy meeting notes with random thoughts]

Output:
## Meeting Notes — January 5, 2026

### Decisions Made:
• Moved deadline to March 15
• Assigned Sarah to lead design phase
• Budget approved at $50K

### Action Items:
• John: Update project timeline by Friday
• Maria: Schedule kickoff meeting
• Team: Review requirements doc

### Open Questions:
• Do we need additional resources?
• When should we schedule the next check-in?
```

### When to Use Organizing Tasks

✓ You have content but it's messy or hard to scan  
✓ You need clear structure for better readability  
✓ You want to convert paragraphs into actionable lists  
✓ You need to make information easy to navigate

## 5. Rewriting Tasks

**Purpose:** Improve clarity, tone, or readability without changing meaning

**What This Includes:**
- Clarity improvements
- Tone adjustments
- Conciseness improvements
- Formality changes
- Readability enhancements

### Common Requests

**Clarity:**
- "Rewrite this to be clearer"
- "Make this easier to understand"
- "Simplify the language"

**Tone:**
- "Rewrite in a more professional tone"
- "Make this more casual"
- "Adjust the tone to be friendlier"

**Conciseness:**
- "Rewrite to be more concise"
- "Remove unnecessary words"
- "Make this tighter"

**Formality:**
- "Make this more formal"
- "Make this less formal"
- "Adjust for a business audience"

### Examples with Context

**Example 1: Improve Clarity**
```
Request: "Rewrite this to be clearer."

Input: "The utilization of our innovative methodology facilitates the optimization of operational efficiency metrics."

Output: "Our method helps you work more efficiently."
```

**Example 2: Adjust Tone**
```
Request: "Rewrite this in a more professional tone."

Input: "Hey! So we're gonna push the deadline back a bit because stuff came up and we need more time, cool?"

Output: "We're extending the deadline to accommodate new requirements that arose during development."
```

**Example 3: Make Concise**
```
Request: "Rewrite to be more concise."

Input: "At this point in time, we are currently in the process of evaluating and assessing the various different options that are available to us."

Output: "We're evaluating our options."
```

### When to Use Rewriting Tasks

✓ The content is good but the wording needs improvement  
✓ You need to adjust tone for a different audience  
✓ The message is buried in unnecessary words  
✓ You want to maintain meaning while improving delivery

## 6. Expanding Tasks

**Purpose:** Add detail, context, or examples without changing core meaning

**What This Includes:**
- Adding examples
- Providing context
- Expanding sections
- Adding explanations
- Including more detail

### Common Requests

**Add Examples:**
- "Add examples to this"
- "Include two examples in this section"
- "Provide an example for each point"

**Add Context:**
- "Add context to this explanation"
- "Include background information"
- "Explain why this matters"

**Expand Sections:**
- "Expand section 2 with more detail"
- "Make this more comprehensive"
- "Develop this point further"

**Add Explanation:**
- "Add a short explanation"
- "Explain how this works"
- "Provide more detail about the process"

### Examples with Context

**Example 1: Add Examples**
```
Request: "Add two examples to this paragraph."

Input: "Good writing is clear and direct."

Output: "Good writing is clear and direct. For example, 'We'll meet at 2 PM' is clearer than 'We should convene at approximately 2 PM in the afternoon timeframe.' Similarly, 'This saves time' is more direct than 'This facilitates temporal efficiency optimization.'"
```

**Example 2: Expand Section**
```
Request: "Expand section 2 with more detail."

Input:
"## Section 2: Benefits
This approach saves time."

Output:
"## Section 2: Benefits
This approach saves time by eliminating unnecessary steps and reducing back-and-forth communication. Instead of multiple revision cycles, users get clean outputs on the first try 90% of the time. This translates to 2-3 hours saved per week per team member."
```

**Example 3: Add Context**
```
Request: "Add a short explanation of why this matters."

Input: "All team members must submit reports by Friday."

Output: "All team members must submit reports by Friday. This deadline ensures we have time to compile results and present to leadership on Monday, which is critical for Q1 planning decisions."
```

### When to Use Expanding Tasks

✓ The content is too brief or lacks detail  
✓ You need examples to illustrate points  
✓ Context is missing and readers may be confused  
✓ You want to make content more comprehensive

## 7. Condensing Tasks

**Purpose:** Reduce length while preserving essential meaning

**What This Includes:**
- Shortening text
- Removing unnecessary content
- Tightening language
- Eliminating redundancy
- Making content more concise

### Common Requests

**Shorten:**
- "Shorten this to one paragraph"
- "Reduce this to 100 words"
- "Make this half as long"

**Remove Unnecessary:**
- "Remove anything unnecessary"
- "Cut the fluff"
- "Keep only essential information"

**Make Concise:**
- "Make this more concise"
- "Tighten this up"
- "Remove redundancy"

### Examples with Context

**Example 1: Shorten to Paragraph**
```
Request: "Shorten this to one paragraph."

Input: [Three paragraphs of repetitive content]

Output: "Our new service helps businesses streamline operations, reduce costs, and improve customer satisfaction. Available now, it includes automated workflows, real-time analytics, and dedicated support. Get started today at [link]."
```

**Example 2: Remove Unnecessary**
```
Request: "Remove anything unnecessary."

Input: "At this current point in time, we are actively working on the process of developing and creating a comprehensive new training program that will help to facilitate and enable better employee performance and productivity outcomes."

Output: "We're developing a new training program to improve employee performance."
```

**Example 3: Make Concise**
```
Request: "Make this more concise without losing meaning."

Input: "The meeting that was scheduled for next week on Tuesday at 3:00 PM in the afternoon has been moved and rescheduled to a new time which is now Thursday at 2:00 PM."

Output: "Tuesday's 3 PM meeting moved to Thursday at 2 PM."
```

### When to Use Condensing Tasks

✓ Content is too long for the medium (email, social post)  
✓ You need to respect word count limits  
✓ Content is repetitive or wordy  
✓ You want to make content more scannable

## 8. Comparison Tasks

**Purpose:** Analyze differences, similarities, or trade-offs between options

**What This Includes:**
- Comparing two or more options
- Highlighting differences
- Identifying strengths and weaknesses
- Creating comparison tables
- Analyzing trade-offs

### Common Requests

**Compare Options:**
- "Compare these two options"
- "What are the differences between A and B?"
- "Compare these approaches"

**Highlight Differences:**
- "Highlight the key differences"
- "What makes these different?"
- "Show me how these differ"

**Identify Strengths:**
- "What are the strengths of each?"
- "Identify pros and cons"
- "Compare benefits and drawbacks"

**Create Tables:**
- "Create a comparison table"
- "Build a side-by-side comparison"
- "Make a feature comparison chart"

### Examples with Context

**Example 1: Compare Options**
```
Request: "Compare these two marketing strategies."

Output:
## Strategy A: Social Media Focus
**Strengths:** Low cost, direct audience engagement, measurable results
**Weaknesses:** Requires consistent content creation, time-intensive
**Best for:** Building community and brand awareness

## Strategy B: Email Marketing
**Strengths:** Higher conversion rates, owned audience, automation possible
**Weaknesses:** Requires email list building, risk of spam perception
**Best for:** Direct sales and customer retention
```

**Example 2: Comparison Table**
```
Request: "Create a comparison table for these project management tools."

Output:
| Feature | Tool A | Tool B | Tool C |
|---------|--------|--------|--------|
| Price | $10/month | $15/month | Free |
| Team size | Up to 10 | Unlimited | Up to 5 |
| Integrations | 50+ | 100+ | 10 |
| Mobile app | Yes | Yes | No |
| Best for | Small teams | Large teams | Solo users |
```

**Example 3: Highlight Differences**
```
Request: "Highlight the key differences between these two proposals."

Output:
**Proposal A** focuses on quick implementation (3 months) with basic features, costing $50K. **Proposal B** takes longer (6 months) but includes advanced features and training, costing $85K. The main trade-off is speed vs. comprehensiveness.
```

### When to Use Comparison Tasks

✓ You're deciding between multiple options  
✓ Stakeholders need to see differences clearly  
✓ You want to identify trade-offs objectively  
✓ You need to present options for decision-making

## 13. Task Combinations

Many real-world requests combine multiple task types:

### Example Combinations

**Draft + Format:**
- "Draft an email and format it with clear sections"

**Summarize + Organize:**
- "Summarize these notes and organize them into sections"

**Rewrite + Condense:**
- "Rewrite this to be clearer and reduce to one paragraph"

**Expand + Format:**
- "Expand section 2 and add bullet points"

**Organize + Assemble:**
- "Organize these pieces and assemble them into a final document"

### How to Request Combinations

**Option 1: Sequential (Recommended)**
1. "Draft a product description"
2. "Now add bullet points for features"
3. "Now shorten to 100 words"

**Option 2: Combined (If Tasks Are Closely Related)**
- "Draft a short product description with bullet-point features"

**Why Sequential Is Often Better:**
- You review output at each step
- Easier to adjust direction
- Clearer what's being requested
- Less chance of confusion

## 15. Quick Reference Guide

### 11 Task Categories

1. **Drafting** — Create new content from scratch
2. **Summarizing** — Condense information
3. **Organizing** — Structure or reorganize content
4. **Rewriting** — Improve clarity, tone, readability
5. **Expanding** — Add detail without changing meaning
6. **Condensing** — Reduce length while keeping meaning
7. **Comparison** — Analyze differences or similarities
8. **Assembly** — Combine pieces into final document
9. **Formatting** — Improve presentation
10. **Clarification** — Identify gaps or improvements
11. **Troubleshooting** — Fix misalignment or reset

### Most Common Requests

**Top 10 Requests:**
1. "Summarize this in bullet points"
2. "Draft an email about [topic]"
3. "Organize this into sections"
4. "Rewrite this to be clearer"
5. "Shorten this to [length]"
6. "Add headings to this"
7. "Turn this into a checklist"
8. "Make this more concise"
9. "Compare these options"
10. "Create a final version"

### Task Selection Tips

**Ask yourself:**
- Am I creating something new? → **Drafting**
- Do I need this shorter? → **Summarizing** or **Condensing**
- Is this messy or hard to read? → **Organizing** or **Formatting**
- Is the wording unclear? → **Rewriting**
- Do I need more detail? → **Expanding**
- Am I choosing between options? → **Comparison**
- Am I combining pieces? → **Assembly**
- Do I need feedback? → **Clarification**
- Is this off track? → **Troubleshooting**
