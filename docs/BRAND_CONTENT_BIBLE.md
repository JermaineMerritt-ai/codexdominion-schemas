### Motion System

**Easing Curves:**
- `ease-in-out`: Smooth transitions (buttons, cards)
- `ease-out`: Exit animations (modals, toasts)
- `ease-in`: Entry animations (fade-ins, slide-ins)

**Durations:**
- **Fast:** 150ms (hover, focus)
- **Standard:** 300ms (cards, buttons, list items)
- **Slow:** 600ms (page transitions, rank-up animations)

**Animation Principles:**
1. **Purposeful:** Every animation has a reason (feedback, hierarchy, delight)
2. **Subtle:** Animations enhance, they don't distract
3. **Accessible:** Respect `prefers-reduced-motion`

**Example (Rank-Up Animation):**
```css
@keyframes rank-up {
  0% { transform: scale(1); }
  50% { transform: scale(1.1); box-shadow: 0 0 20px gold; }
  100% { transform: scale(1); }
}

.rank-up-animation {
  animation: rank-up 0.6s ease-in-out;
}

@media (prefers-reduced-motion: reduce) {
  .rank-up-animation {
    animation: none;
  }
}
```

### Accessibility Rules

**WCAG 2.1 AA Compliance:**

1. **Color Contrast:**
   - Body text: 7:1 ratio (AAA standard)
   - UI elements: 4.5:1 ratio minimum
   - Large text (18px+): 3:1 ratio minimum

2. **Focus Indicators:**
   - Visible focus ring on all interactive elements
   - 2px solid outline, offset by 2px
   - High contrast (gold on dark, obsidian on light)

3. **Keyboard Navigation:**
   - All interactive elements accessible via Tab
   - Enter/Space activates buttons
   - Arrow keys for lists and menus

4. **Screen Readers:**
   - Semantic HTML (header, nav, main, article, footer)
   - ARIA labels for icons and complex UI
   - Alt text for all images

5. **Motion:**
   - Respect `prefers-reduced-motion`
   - No auto-playing videos
   - Pause/stop controls for animations

---

### Content Rules

**Universal Principles:**

1. **Short Sentences**
   - Target: 15-20 words max
   - Break complex ideas into multiple sentences
   - One idea per sentence

2. **Clear CTAs**
   - Action verbs: Start, Create, Share, Earn, Buy, Support
   - Specific outcomes: "Start earning today" > "Get started"
   - No ambiguity

3. **No Jargon**
   - Avoid: "Leverage your digital assets"
   - Use: "Sell your products"
   - If technical term needed, explain in tooltip

4. **Always Human**
   - Write like you're talking to a friend
   - Avoid corporate speak
   - Use "you" and "your"
   - Contractions are fine (we're, you're, it's)

5. **Culturally Grounded**
   - Caribbean voice: warm, confident, direct
   - Celebrate heritage without stereotypes
   - Global-ready but locally authentic
   - No forced patois or slang (be natural, not performative)

---

### Voice by Content Type

**Emails:**
- Subject line: Clear, benefit-driven, no clickbait
- Body: Short paragraphs (2-3 sentences max)
- CTA: One primary CTA per email
- Sign-off: Warm but not overly familiar

**Social Posts:**
- Instagram: Visual-first, short captions (1-2 sentences)
- TikTok: High energy, trend-aware, youth-focused
- YouTube: Educational, value-driven, longer-form
- LinkedIn: Professional, thought leadership, data-driven

**UX Micro-Copy:**
- Buttons: Action verbs ("Upload Product", "Share Link")
- Errors: Clear + actionable ("File too large. Max 10MB.")
- Success: Celebratory + next step ("Product live! Share your link.")

## 5. PRODUCT FLOWS

### Core User Journeys

All UX flows are defined in [FRONTEND_REQUIREMENTS.md](FRONTEND_REQUIREMENTS.md) and implemented in [dashboard-app/](../dashboard-app/).

**Summary:**

1. **Sign Up**
   - Creator, youth, or buyer

2. **Upload Product (Creator)**
   - Fill product form (title, description, price, file)
   - Publish or save as draft
   - Get shareable link

3. **Share Link (Youth)**
   - Copy unique referral link
   - Share on social, email, SMS
   - Track clicks and earnings

4. **Purchase (Buyer)**
   - Browse marketplace
   - Add to cart
   - Checkout (Stripe)
   - Download product

5. **Payout (Creator + Youth)**
   - View available balance
   - Request payout (PayPal, bank, Stripe, Venmo, CashApp)
   - Track status (pending ‚Üí processing ‚Üí completed)

6. **Leaderboard (Youth)**
   - View weekly/monthly rankings
   - See current rank and earnings
   - Earn badges (top-10, high-roller, first-sale, etc.)
   - Track rank changes (rank-up animation)

7. **Challenges (Youth)**
   - Join DominionYouth challenge
   - Complete challenge goal (e.g., "First Sale")
   - Earn prizes ($50 bonus, badges)
   - Climb leaderboard

8. **Onboarding**
   - Creator: Upload first product ‚Üí Publish ‚Üí Share link
   - Youth: Get referral link ‚Üí Share on social ‚Üí Make first sale
   - Buyer: Browse products ‚Üí Purchase ‚Üí Support creators

---

### Micro-States (Error, Loading, Success)

**Upload Product:**
- **Default:** Empty form
- **Loading:** "Uploading..." progress bar
- **Success:** "Product live! Share your link." + shareable link displayed
- **Error:** "Upload failed. File too large (max 10MB)."

**Share Link:**
- **Default:** "Copy Link" button
- **Clicked:** Button text changes to "Copied!" for 2 seconds
- **Error:** "Unable to copy. Please try again."

**Purchase:**
- **Default:** "Buy Now" button
- **Loading:** Spinner on button, text: "Processing..."
- **Success:** Redirect to download page + success toast
- **Error:** "Payment failed. Please try again." + retry button

**Payout Request:**
- **Default:** "Request Payout" button
- **Loading:** "Submitting..." spinner
- **Success:** "Payout requested! We'll process it within 3-5 days."
- **Error:** "Insufficient balance. You need $10 minimum to withdraw."

**Leaderboard Rank Change:**
- **Rank Improved:** Rank-up animation + confetti + toast "üéâ You moved up to rank #15!"
- **Rank Dropped:** No animation, just updated number
- **New Badge:** Badge reveal animation + tooltip "High Roller Unlocked!"
