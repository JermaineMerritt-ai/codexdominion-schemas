# Template Structure

Standardized component structure for CodexDominion constellation.

## Files

- `index.ts` - Main entry point with exports
- `routes.ts` - API route definitions
- `service.ts` - Business logic and services
- `tests/` - Unit test suite

## Usage

Copy this template structure when creating new:
- Apps (sovereigns)
- Packages (custodians)
- Agents (avatars)
- Core modules

## Standards

Follow naming conventions:
- Variables: camelCase
- Classes: PascalCase
- Files: kebab-case
